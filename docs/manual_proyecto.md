# 📊 Manual del Proyecto – THE FORUM

## 1. Introducción

El presente documento describe el desarrollo del proyecto THE FORUM a lo largo de los dos trimestres del curso.

Incluye las fases realizadas, decisiones tomadas y evolución del sistema desde su planteamiento inicial hasta su estado actual.

---

## 2. Descripción del proyecto

THE FORUM es una aplicación web orientada a la creación de foros temáticos, inicialmente centrados en videojuegos, con posibilidad de expansión a otros ámbitos.

El objetivo principal es ofrecer una plataforma organizada y funcional donde los usuarios puedan interactuar mediante publicaciones y debates.

---

## 3. Fases del proyecto

### 3.1 Presentación inicial

En la fase inicial se realizó:

- Presentación de la asignatura
- Planteamiento de la idea del proyecto
- Definición general del alcance

---

### 3.2 Imagen corporativa

Se diseñó la identidad del proyecto, incluyendo:

- Nombre: THE FORUM
- Concepto visual
- Enfoque de la aplicación

---

### 3.3 Recogida de requisitos

Se elaboró un documento de requisitos tras simular la interacción con una empresa cliente.

#### Requisitos funcionales principales:
- Registro e inicio de sesión
- Creación de foros
- Publicación de mensajes
- Gestión de usuarios

#### Requisitos no funcionales:
- Compatibilidad con navegadores
- Diseño responsive
- Facilidad de uso

---

### 3.4 Diseño de interfaces

Se desarrollaron prototipos de la interfaz de usuario:

- Página principal
- Vista de foros
- Vista de publicaciones
- Formularios de login y registro

El objetivo fue garantizar una experiencia de usuario clara e intuitiva.

---

### 3.5 Diseño de base de datos

Se diseñó la estructura de datos mediante:

- Modelo Entidad-Relación (MER)
- Modelo relacional

#### Entidades principales:
- Usuarios
- Roles
- Videojuegos
- Foros
- Publicaciones

Se establecieron relaciones claras y escalables entre las entidades.

---

### 3.6 Presentación a la empresa

Se realizó una presentación del proyecto donde se mostraron:

- Interfaces diseñadas
- Estructura de base de datos

---

### 3.7 Elección de tecnologías

Inicialmente se planteó el uso de:

- AngularJS
- Bootstrap

Posteriormente se decidió migrar a tecnologías más modernas:

- React
- Tailwind CSS
- Supabase

#### Motivos del cambio:
- Mejor rendimiento
- Mayor flexibilidad
- Mejor adaptación a tendencias actuales

---

### 3.8 Estructuración de la documentación

Se definió la organización de la documentación del proyecto:

- Manual de usuario
- Manual técnico
- Manual de despliegue
- Manual del proyecto

---

### 3.9 Desarrollo de manuales

Se comenzó el desarrollo de:

- Manual de usuario
- Manual técnico

Con el objetivo de facilitar tanto el uso como el mantenimiento del sistema.

---

### 3.10 Estudio de despliegue

Se analizaron distintas opciones de despliegue:

- Entorno local
- Servidores web
- Plataformas cloud

Se realizaron pruebas iniciales en entorno local.
